# Do : 하다 — 베타 검증 시나리오 (QA Test Scenarios)

> **대상**: 여름방학 단기 검증 인력(알바 테스터)
> **목표**: 설치(Android·iOS) → 가입 → 하다 생성 → 인증 → 동료·응원 → 완주 → 완주 이야기 공유 → 회원 탈퇴까지 **앱 전체 기능을 처음부터 끝까지 빠짐없이** 검증한다.

## 📖 0. 테스터 오리엔테이션 (검증 시작 전 필독)

### 0.1 이 앱이 뭐예요? (5줄 요약)
- **"하다(Do:하다)"** = 비교·자랑 대신 **"되어가는 과정"을 동료와 함께 목격하는** 조용한 SNS.
- 사용자가 **"하다"(=도전/챌린지)** 를 만들고, 매일(또는 목표 개수만큼) **인증(사진)** 을 올린다.
- 4가지 방 종류: **나홀로 · 응원받기 · 다함께 · 누구나**. 종류마다 경험이 다르다.
- 좋아요·팔로워·랭킹 없음. 대신 **응원 · 댓글 · 4가지 평가 · 연속 메달 · 박제(완주 기록 영구 보존)**.
- 목표를 다 채우면 **완주 → 박제 → "해냈어요(완주 이야기)" 공유**.

### 0.2 마음가짐 (중요)
- **"내가 이상하게 써서 그런가?" 하지 말 것.** 이상하게 쓰다 깨지면 그게 버그다. 일부러 비틀어서 써보라.
- **느린 것·버벅임·빈 화면·이상한 한국어·어색한 띄어쓰기·잘린 글자**도 전부 리포트 대상이다.
- **될 것 같은데 안 되는 것** > "안 될 것 같아서 안 해본 것". 끝까지 눌러봐라.
- 캡처(스크린샷·화면녹화)를 **습관처럼** 남겨라. 말로 설명한 버그는 못 고친다.

### 0.3 준비물
- [ ] 테스트용 스마트폰 (가능하면 Android·iPhone 둘 다, 기종·OS 버전 다양할수록 좋음)
- [ ] Google 계정 (테스트 전용 권장) — Apple 로그인 검증 시 Apple ID
- [ ] 와이파이 + LTE/5G (네트워크 전환 테스트용)
- [ ] **2명 이상 한 조** — 초대·합류·응원·댓글·내기 등은 혼자 검증 불가 (계정이 2개 이상 필요)
- [ ] 버그 기록 시트(아래 0.6 템플릿) — 스프레드시트 권장

### 0.4 설치/배포 채널 (운영자가 채워서 배포)
| 구분 | 방법 | 비고 |
|---|---|---|
| Android | _(Play 내부 테스트 링크 또는 APK)_ | 운영자 제공 |
| iOS | _(TestFlight 초대 링크)_ | 운영자 제공 · Apple ID 필요 |
| OTA 업데이트 | 앱 재시작 시 자동 | 프로필 하단 버전·updateId 로 확인 |

### 0.5 심각도(우선순위) 기준
| 등급 | 의미 | 예시 |
|---|---|---|
| **S1 (치명)** | 앱이 죽거나, 더 진행 불가, 데이터 손실/유출 | 로그인 후 흰 화면, 인증 누르면 강제종료, 남의 정보 노출 |
| **S2 (심각)** | 핵심 기능이 안 됨, 우회로도 없음 | 하다 생성 실패, 인증 사진 업로드 실패, 완주 판정 안 됨 |
| **S3 (보통)** | 기능은 되나 잘못 동작/오표시 | 인원수 틀림, 알림 안 옴, 날짜 하루 밀림 |
| **S4 (경미)** | 표시/문구/디자인 문제 | 글자 잘림, 어색한 한국어, 여백 깨짐, 오타 |

### 0.8 ⏱️ "빠른 완주" 만들기 팁 (완주·박제·해냈어요·내기 정산 검증용)
완주까지 며칠을 기다릴 수 없으므로 **테스트 전용으로 짧게** 만든다.
- **목표 횟수형(🎯 count) + 목표 1개** → 인증 1번이면 **즉시 완주**. (가장 빠름)
- **주기형 + 기간 1일 + 매일** → 오늘 1번 인증으로 당일 완주.
- 내기/응원 정산은 완주가 떠야 "받기/기부"가 열리므로 위 방법으로 빠르게 완주 상태를 만든다.

# 🅰️ Phase A — 설치 · 첫 실행 · 권한

### TC-A1. 앱 설치 (Android / iOS)
- 🎯 목적: 배포 채널에서 정상 설치되는가
- 👥 1인 / Android·iOS 각각
- 📋 절차
  1. 운영자가 준 링크로 앱 설치 (Play 내부테스트 / TestFlight / APK)
  2. 홈 화면에 **"Do:하다"** 아이콘 + 이름이 정상 표기되는지 확인
  3. 앱 실행
- 🟢 기대: 설치 성공, 아이콘·앱이름 정상, 첫 실행 시 스플래시 진입
- ⚠️ 확인: 아이콘 깨짐/저화질, 앱 이름 오타, 설치 중 에러, iOS "신뢰되지 않은 개발자" 안내

### TC-A2. 권한 요청 (카메라·사진·알림)
- 🎯 목적: 권한 요청 타이밍·문구·거부 후 동작
- 📋 절차
  1. 인증/기록에서 **카메라·사진 보관함** 권한 요청이 뜨는 시점 확인
  2. 알림 권한 요청 시점 확인
  3. **일부러 "거부"** 를 눌러본 뒤, 다시 인증을 시도
- 🟢 기대: 거부해도 앱이 죽지 않음. 다시 시도하면 설정 안내 또는 재요청. 문구가 한국어로 자연스러움
- ⚠️ 확인: 권한 거부 시 강제종료, 권한 설명 문구 없음/영어, 알림 권한 안 물어봄

### TC-A3. 콜드/웜 스타트 & 백그라운드 복귀
- 📋 절차
  1. 앱 완전 종료 후 재실행(콜드 스타트) → 로딩 시간·흰 화면 체크
  2. 홈 메뉴(앱 전환)로 백그라운드 → 다시 복귀
  3. 화면 잠금 후 해제 → 복귀
- 🟢 기대: 어느 경우든 마지막 화면 또는 홈으로 자연 복귀, 세션 유지(재로그인 요구 X)
- ⚠️ 확인: 복귀 시 흰 화면/무한로딩, 로그인 풀림, 입력 중이던 내용 날아감

# 🅱️ Phase B — 온보딩 · 로그인 · 약관

### TC-B1. 스플래시 & 온보딩 스와이프
- 📋 절차
  1. 첫 실행 → 스플래시(주황 배경, `( ⊙ )` 마크, "Do : 하다", "더 나은 나, 더 나은 세상")
  2. 화면 탭 → 온보딩(onb1~onb4) 진입
  3. **좌우로 스와이프(플링)** 하며 4장 넘기기
  4. 스플래시 하단 **"건너뛰기"** 로 온보딩 생략 → 로그인 직행도 확인
- 🟢 기대: 스와이프 부드럽게 4장, 마지막 장에서 로그인/시작 진입, 건너뛰기 동작
- ⚠️ 확인: 스와이프 안 먹힘, 인디케이터(점) 안 맞음, 글자 잘림, 뒤로가기 시 갇힘

### TC-B2. Google 로그인 (신규 가입)
- 🎯 핵심 — **자동 테스트 의무 영역(인증/권한)**, 가장 중요
- 📋 절차
  1. 로그인 화면 → **"Google로 시작하기"**
  2. 구글 계정 선택 → 권한 동의
  3. 신규 사용자면 **환영(welcome) 화면 → 약관 동의 → 홈** 흐름 확인
- 🟢 기대: 로그인 성공 후 신규는 약관, 기존은 홈으로
- ⚠️ 확인: 로그인 취소 시 에러 팝업 없이 복귀, 무한 로딩, "로그인 실패" 후 재시도 가능 여부

### TC-B3. Apple 로그인 (iOS 전용)
- 📋 절차
  1. **iOS 기기**에서 로그인 화면에 **"Apple로 시작하기"** 검정 버튼이 보이는지
  2. Apple ID로 로그인 (이메일 공유/숨김 모두 시도)
- 🟢 기대: Apple 로그인 성공 → 신규는 약관, 기존은 홈. 취소 시 조용히 복귀
- ⚠️ 확인: Android엔 Apple 버튼이 **안 보여야** 정상. iOS에서 버튼 누락 시 S1(앱스토어 정책)

### TC-B4. 약관 동의 화면
- 📋 절차
  1. 신규 가입 후 환영 화면 진입
  2. **[필수]** 만 14세 이상 / 서비스 이용약관 / 개인정보 처리방침, **[선택]** 마케팅 수신 확인
  3. 필수만 체크 → "시작하기" 활성화 확인
  4. "전체 동의" 토글, "보기" 링크 동작 확인
- 🟢 기대: 필수 3개 체크 전엔 "시작하기" 비활성. 체크 후 활성→홈
- ⚠️ 확인: 비활성인데 눌러짐, "보기" 링크 죽어있음, 약관 본문 없음

### TC-B5. 자동 로그인(세션 유지) & 로그아웃 후 재로그인
- 📋 절차
  1. 로그인 상태에서 앱 종료 → 재실행 → 스플래시가 **자동으로 홈** 진입하는지
  2. 프로필 → 로그아웃 → 다시 로그인
- 🟢 기대: 세션 복원되면 온보딩 없이 홈 직행. 로그아웃 후 재로그인 정상
- ⚠️ 확인: 매번 로그인 요구, 로그아웃이 안 됨

# 🅲 Phase C — 홈 첫 화면 (콜드스타트 온램프)

### TC-C1. 신규(빈) 사용자 홈 — 온램프
- 📋 절차
  1. 가입 직후(도전 0개) 홈 화면 관찰
  2. **"🔭 하다 구경"** hero, **"누구나 합류"** 섹션, 선언/구경/합류 안내가 보이는지
- 🟢 기대: 텅 빈 게 아니라 시작 갈래(선언·구경·합류)가 안내됨
- ⚠️ 확인: 완전 빈 화면, "응원받기"만 강요(올인) 느낌, CTA 죽어있음

### TC-C2. 활동 사용자 홈 — SNS 피드
- 📋 절차
  1. 도전·동료가 있는 상태에서 홈 관찰
  2. **me-strip(상단 내 요약 1줄)** + 피드 카드 5종 확인:
     🎉 완주리본 · 📸 오늘인증 · 🙋 응원받기 · 🌍 누구나합류 · ✨ 관심도전
  3. 맨 아래 **🌙 끝 마커** 확인
  4. 미인증 우선 노출 / "모두 보기" 링크 / 미인증 선택 모달 동작
- 🟢 기대: 카드가 분류별로 자연스럽게, 무한스크롤 아닌 **끝이 있는** 피드

# 🅴 Phase E — 하다 만들기 (생성 마법사 5단계)

> ＋ 버튼(가운데 큰 버튼) → 5단계: **① 제목 ② 분류 ③ 방 타입 ④ 기간+빈도(또는 목표개수) ⑤ 인증 방식**
> **4가지 방 타입 × 2가지 목표유형(주기형/목표횟수형)** 을 조합해 빠짐없이 만들어 본다.

### TC-E1. 마법사 공통 흐름 & 검증 게이트
- 📋 절차
  1. ＋ → Step1 제목(2글자 미만이면 "다음" 비활성 확인)
  2. Step2 분류(대분류 → 소분류)
  3. Step3 방 타입(나홀로/응원받기/다함께/누구나, 기본=응원받기)
  4. Step4 기간 + 빈도(주기형) 또는 목표 개수(목표횟수형)
  5. Step5 인증 방식(기본 사진)
  6. "← 이전"/"취소", 단계 인디케이터 확인
- 🟢 기대: 각 단계 미충족 시 "다음" 비활성, 뒤로가기 자연스러움
- ⚠️ 확인: 비활성인데 넘어감, 뒤로가면 입력 초기화, 단계 표시 틀림

### TC-E2. 나홀로(solo) 하다 생성
- 📋 절차: 방타입=나홀로, 주기형 매일 3일 정도로 생성
- 🟢 기대: 생성 후 방 진입. **나홀로 방은 알림 0건·다짐 가능·내기 self만**. 시작일 선택 없음
- ⚠️ 확인: 나홀로인데 모집/초대 노출, 생성 실패

### TC-E3. 응원받기(cheered) 하다 생성 — 기본값
- 📋 절차: 방타입=응원받기(기본), 안내문(소개글+이미지) 입력, 생성
- 🟢 기대: 생성 후 초대 가능. **나만 인증 주체, 나머지는 응원만**. 인포바 "📸 1/1"(도전자 1명 기준)
- ⚠️ 확인: 응원받기인데 "전원 인증"처럼 표기, 안내문/이미지 누락

### TC-E4. 다함께(closed) 하다 생성 — 시작일·모집기간
- 📋 절차
  1. 방타입=다함께, **시작일을 '내일~+7일'** 중 선택, 안내문 입력
  2. 생성 후 방 진입
- 🟢 기대: 시작 전이면 **"🗓️ N일 뒤 시작 · 동료 모집 중"** + "시작 D-N". 인증 불가, 합류·대화는 가능
- ⚠️ 확인: 시작 전인데 인증됨, 날짜가 하루 밀림(오늘이 비활성), 시작일 리셋 안 됨

### TC-E5. 누구나(open) 하다 생성 — 공개·모집
- 📋 절차: 방타입=누구나, 시작일·안내문 입력, 생성
- 🟢 기대: 생성 후 **하다 구경·둘러보기·홈 누구나합류**에 노출(모집 중일 때만)
- ⚠️ 확인: 비공개인데 노출, 노출 안 됨

### TC-E6. 목표 횟수형(count) 하다 생성
- 📋 절차
  1. Step4에서 유형 토글 **🎯 목표 횟수형** 선택
  2. 목표 개수 입력(예: 3) + 프리셋 칩
  3. 생성
- 🟢 기대: "기간 내 N개 달성" 안내. **하루 다회 인증 OK · 다 채우면 즉시 완주**. **내기 스텝 스킵(count는 내기 비활성)**
- ⚠️ 확인: count인데 "매일 인증" 잔소리 노출, 목표 0개 허용, 내기 노출

### TC-E7. AI 콘텐츠 검수 (생성 시) — 우회 없음
- 🎯 **자동 테스트 의무 영역(AI 검수)** — 손으로도 확인
- 📋 절차
  1. 제목/안내문에 **부적절한 내용(욕설·성적·폭력·불법)** 을 넣고 생성 시도
  2. 정상 내용으로 다시 생성
- 🟢 기대: 부적절 내용은 **차단(거부)**, 정상은 통과. 검수 건너뛰는 백도어 없음
- ⚠️ 확인: 부적절 통과, 정상인데 과민 차단(특히 책/감상 등), 검수 중 무한 로딩

### TC-E8. 1일 도전 자동 보정 & 방타입 변경 시 리셋
- 📋 절차: 기간 1일로 설정 → 방타입이 다함께로 자동 보정되는지, 방타입 바꿨을 때 시작일/내기 설정 리셋되는지
- 🟢 기대: 1일은 자동 다함께, 방타입 변경 시 미래 시작일·내기 설정 정리
- ⚠️ 확인: 옛 시작일/내기 설정 잔존

### TC-G1. 카메라로 인증

  2. 카드에서 **좌우 스와이프**, 탭하면 전체화면 **좌우 스와이프 + 핀치줌**
- 🟢 기대: 3장까지 등록, 캐러셀 점·"N/M" 배지, 전체화면 페이징·줌 정상
- ⚠️ 확인: 4장 이상 선택됨, 캐러셀 안 넘어감, 전체화면 줌 중 페이징 꼬임(실기기 필수)

### TC-G3. 연속 인증 메달(streak)
- 📋 절차: 연속으로 며칠(또는 백필된 방) 인증 → 3·7·21일 등 **마일스톤** 게시글에 오각형 메달이 붙는지
- 🟢 기대: 마일스톤 달성 인증에만 메달, 같은 날 2번째 인증은 메달 X
- ⚠️ 확인: 매 인증마다 숫자 노출(정체성 위반), 메달 색/숫자 틀림

### TC-G4. 인증 제약 (엣지)
- 📋 절차
  1. **시작 전(모집기간)** 방에서 인증 시도 → 차단되는지
  2. **종료/포기/모집마감** 방에서 인증 시도
  3. **응원자**(cheered 비개설자)가 인증 시도 → 불가(응원만)
  4. count형: 하루 2번 인증 → 둘 다 인정되는지
- 🟢 기대: 각 제약이 정확히 동작, 응원자는 인증 주체 아님
- ⚠️ 확인: 막혀야 할 인증이 됨, count 하루 다회가 막힘

### TC-G5. 연속 인증 메달(Streak) 마일스톤 외 날짜 미노출 검증
- 🎯 목적: 마일스톤 날짜(3일, 7일, 21일 등) 외의 일반 날짜에는 메달이 노출되지 않는지 확인
- 📋 절차
  1. 연속 2일차 또는 4일차 등 마일스톤에 해당하지 않는 날에 인증을 진행
  2. 해당 게시글에 Streak 메달 아이콘이나 불필요한 누적 표시가 뜨지 않는지 확인
- 🟢 기대: 3, 7, 21일 등 지정된 핵심 마일스톤 달성 인증글에만 메달이 표출되며 일반 일자에는 노출되지 않음
- ⚠️ 확인: 일반 연속 인증일(예: 2일, 4일)에도 메달이 무차별적으로 뜨거나 숫자가 오표기됨

   - **조치**: [0055 마이그레이션](file:///e:/dohada/supabase/migrations/0055_restore_browse_challenges_finished_visibility.sql)을 작성·적용하여 종료방 필터를 복구하고, "과거 진행된 도전을 둘러보는 히스토리 익명 라이브러리" 용도로 노출한다는 한국어 주석 설명을 추가했습니다.

2. **[S3] 회원 탈퇴 시 `blocks` 테이블 내 탈퇴자의 양방향 차단 레코드 완전 삭제**
   - **문제**: 탈퇴 처리(`delete-account` Edge Function) 시 본인이 차단한 내역(`blocker_id = uid`)만 지우고 있어, 타인이 탈퇴자를 차단했던 내역이 관계 정보로 유실되지 않고 DB에 잔존했습니다.
   - **조치**: Edge Function 코드를 수정하여 탈퇴한 유저와 연관된 모든 차단 레코드를 양방향(`blocker_id = uid OR blocked_id = uid`)으로 안전하게 소거했습니다.

3. **[S4·UX] 회원 탈퇴 API 실패 시 세션 강제 정리 보장**
   - **문제**: Edge Function 호출 예외 발생 시 클라이언트 단에서 `signOut`이 스킵되어 로그인 세션이 꼬이는 고립 상황이 발생할 위험이 있었습니다.
   - **조치**: 클라이언트 `deleteAccount` API 함수 내에 `try-finally` 블록을 도입해, 서버 호출 성공 여부와 무관하게 로컬 로그인 세션은 무조건 강제 파괴하고 로그인 화면으로 리다이렉트되도록 안정성을 높였습니다.

4. **[S4·UX] AI UGC 검수 통신 에러 발생 시 한국어 친화 에러 메시지 래핑**
