> 이 둘은 출시 직전 **무조건 구현**. 기능 품질과 무관하게 없으면 심사에서 막힌다. 상세 설계는 아래 "🧰 착수 준비 메모".

2. ✅ **[4] 회원 탈퇴** — 계정삭제 진입점 + `delete-account` EF. 하드삭제 금지(FK cascade=동료 박제 파괴) → **익명화 + auth ban**. _구글·애플 의무._ (0053·EF·OTA 배포 완료 2026-06-18)

**설정만 하면 되는 것(출시 시 함께)**: [3] EXIF=다음 `eas build` · [5] 크래시=Sentry DSN(.env) · [7] 푸시=APNs/FCM 자격증명.
**남은 순수 개발**: [6] 트래킹(SDK) · [8] ErrorBoundary·netinfo·업로드 재시도. **[1] 콜드스타트**=시드 SQL 실행만(운영계정·목록 결정 후).

> 🔒 모든 항목은 2026-06-14 설계/준비 완료. **결정·구현은 출시 시점에** (아래 메모대로 풀면 됨).

### ✅ 출시 차단 모두 해소 — ~~2(신고·차단)~~ · ~~4(회원탈퇴)~~ 둘 다 배포 완료
스토어 심사 자체를 막는 항목. 신고·차단(2)은 0047·3a·3b, 회원탈퇴(4)는 0053·delete-account EF·OTA(2026-06-18)로 **출시 차단 0개.** 남은 건 설정(EXIF 빌드·Sentry DSN·APNs/FCM)·순수개발(트래킹·ErrorBoundary)·시드뿐.

### 🤖 Claude 작업 — 권장 순서 (1→2→4→3→5→6→7→8)
1. **콜드스타트**: 시드 도전 seed 스크립트(`scripts/seed.ts`/seed SQL) + 홈 0인연 fallback(추천/시드 노출). *(온보딩·빈상태·인연 로직은 완료)*
2. **신고·차단**: `reports`·`blocks` 마이그레이션 + 신고 모달(사유)·유저 차단(양방향 필터) UI + 누적 자동 숨김. 댓글/기록 AI검수 확장.
3. **회원탈퇴**: MY→설정→계정삭제(2단계) + 삭제 Edge Function(유저데이터+Storage 사진) + 잔여 콘텐츠 익명화.
4. **EXIF**: 코드 완료 — **추가 작업 없음, 다음 `eas build`만 하면 활성.**
5. **크래시**: 코드 완료 — `.env`에 `EXPO_PUBLIC_SENTRY_DSN`만(🧑‍💻). 회귀 시나리오 문서화는 🤖 가능.
6. **트래킹**: PostHog/Amplitude SDK 연동 + 핵심 이벤트(가입·도전선언·인증·인연·해냈어요) 발화.
7. **푸시**: 코드 완료 — APNs/FCM 자격증명만(🧑‍💻).
8. **오프라인·에러**: React ErrorBoundary + NetInfo 네트워크 감지 + 업로드 실패 재시도/큐잉.

### 🧑‍💻 선결정 (🤖 작업의 선행조건)
시드 도전 카피/이미지 · 신고 사유 카테고리 · 차단 후 정책 · 삭제 정책(즉시/유예) · 추적 지표 · 알림 종류·빈도 · 에러 카피 · Sentry DSN 발급 · APNs/FCM 키.

## A. 출시를 막는 필수 항목

### ✅ 1. 콜드 스타트 설계 (SNS 최대 리스크)
신규 가입자가 첫 화면에서 빈 피드를 보면 즉시 이탈한다. "남들의 여정"이 중심인 두하다는 더 치명적. **안티 비교 철학을 깨지 않으면서** 화면을 채우는 게 핵심.

- [ ] **시드 도전 준비** (카테고리별 10~20개)
  - 🧑‍💻 매슬로우 단계별로 진짜 따라 하고 싶은 도전 기획 (예: "하루 10분 산책 7일", "감사일기 7일"). 카피·대표 이미지·기간 결정
  - 🤖 Supabase seed 스크립트 작성 → 운영자 계정으로 시드 도전 + 더미 인증 몇 건 입력 (`seed_challenges.sql` 또는 `scripts/seed.ts`)
- [ ] **첫 화면에 "진행 중인 도전"이 보이게**
  - 🤖 홈 피드 쿼리에 fallback 추가: 유저 인연이 0건이면 추천/시드 도전·인증을 노출
  - 🧑‍💻 "추천"으로 보여줄 기준 결정 (최신순? 인증 활발순? 안티-랭킹 원칙 위반 안 되게)
- [ ] **온보딩 플로우** (가입 직후 첫 도전 선언까지)
  - 🧑‍💻 온보딩 3~4단계 카피·플로우 확정 (관심 카테고리 선택 → 첫 도전 추천 → 선언)
  - 🤖 온보딩 화면 구현 + "건너뛰기"해도 막히지 않는 경로
- [ ] **빈 상태(Empty State) UI** — 피드·내 도전·인연·도전방 각각
  - 🤖 빈 상태 컴포넌트 (안내 문구 + 행동 유도 버튼). 에러처럼 안 보이고 "다음에 뭘 할지"가 명확하게
  - 🧑‍💻 각 빈 상태 카피 결정
- [ ] **첫 인연 형성 경로 확보**
  - 🧑‍💻 유저가 적을 때도 같은 도전에 참여하면 인연이 맺어지는지 시나리오 점검
  - 🤖 인연 형성 로직이 "동일 도전 참여" 기준으로 동작하는지 확인·수정

> 🔑 우선순위 1순위. 시드 데이터 없이 출시하면 마케팅으로 데려온 유저가 다 빠져나간다.

---

### ✅ 2. 신고·차단 기능 (UGC 필수)
없으면 **애플 심사 반려 + 법적 리스크** 둘 다 걸린다. Supabase 모더레이션 테이블은 이미 있으니, 실제 동작하는 UI·플로우 완성이 목표.

- [ ] **콘텐츠 신고** (인증 사진·댓글·도전 등)
  - 🧑‍💻 신고 사유 카테고리 확정 (예: 스팸/욕설/음란물/폭력/사칭/기타)
  - 🤖 신고 버튼 + 사유 선택 모달 → `reports` 테이블 insert. 중복 신고 방지
- [ ] **유저 차단**
  - 🤖 `blocks` 테이블 + 차단 시 해당 유저의 콘텐츠를 피드·도전방·인연에서 양방향 필터링
  - 🧑‍💻 차단 후 동작 정책 (차단 알림 안 보냄, 같은 도전방에서 콘텐츠만 숨김 등)
- [ ] **자동 모더레이션** (Claude Haiku 텍스트)
  - 🤖 도전·댓글 등록 시 Haiku 텍스트 체크 → 위험 점수 높으면 보류/플래그
  - 🧑‍💻 차단 임계값·차단 후 처리 정책 (자동 숨김 vs 운영자 확인)
- [ ] **이미지 신고 큐** (사진 인증 대비)
  - 🤖 신고된 이미지가 모이는 큐 (별도 화면 없이 Supabase 콘솔로 갈음 가능)
  - 🧑‍💻 신고 처리 SLA 결정 (몇 시간 내 확인, 누가 처리할지)
- [ ] **노출 즉시 차단 옵션**
  - 🤖 신고 누적 N건 시 자동 노출 중단 로직

> 🔑 애플 가이드라인 1.2 (UGC) 충족의 핵심. 신고·차단·필터링·연락수단 4가지를 본다.

## B. 출시 품질 항목

### ✅ 3. 인증 사진 EXIF 제거
사진 인증이 핵심 기능. 업로드 사진에 GPS 등 메타데이터가 남으면 위치가 노출된다.

- [ ] **업로드 전 EXIF 스트립**
  - 🤖 `expo-image-manipulator`로 리사이즈/재인코딩 → 메타데이터 자동 제거. Supabase Storage 업로드 직전 파이프라인에 삽입
  - 🤖 처리 실패 시 원본 업로드를 막는 가드 (메타데이터 남은 채 올라가는 사고 방지)
- [ ] **이미지 최적화**
  - 🧑‍💻 최대 해상도·용량 정책 결정 (예: 장변 1080px, 80% 품질)
  - 🤖 위 정책대로 압축 적용 (스토리지·전송 비용 절감)

---

### ✅ 4. 회원 탈퇴 & 데이터 삭제
구글·애플 둘 다 **앱 내 계정 삭제**를 의무화. 없으면 심사 반려.

- [ ] **앱 내 탈퇴 진입점**
  - 🤖 MY 탭 → 설정 → 계정 삭제 경로 구현 (2단계 확인)
- [ ] **데이터 삭제 처리**
  - 🧑‍💻 삭제 정책 결정: 즉시 삭제 vs 유예기간(예: 30일 후 영구삭제), 법적 보관 의무 데이터 여부
  - 🤖 Supabase RPC/Edge Function으로 유저 데이터 삭제 + Storage의 인증 사진도 함께 삭제
  - 🤖 soft delete 택할 경우 모든 조회 쿼리에서 삭제 유저 제외
- [ ] **탈퇴 후 부수 처리**
  - 🤖 탈퇴 유저가 남긴 도전방·인연 콘텐츠 처리 (작성자 익명화 등)

---

### ✅ 5. 크래시 리포팅 & 핵심 플로우 안정성
- [ ] **크래시 추적 도입**
  - 🤖 `sentry-expo` 통합 + 릴리스 버전 태깅
- [ ] **핵심 플로우 E2E 점검** (실기기)
  - 🧑‍💻 가입 → 도전 선언 → 사진 인증 → 인연 형성 → 해냈어요 전 구간을 iPhone 12 Pro Max + 안드로이드 실기기에서 끝까지 통과
  - 🤖 점검 시나리오 체크리스트 문서화 (회귀 테스트용)
- [ ] **베타 버그 정리**

### ✅ 7. 푸시 알림 (리텐션 핵심 레버)
- [ ] **알림 인프라**
  - 🤖 `expo-notifications` 설정 + 권한 요청 흐름 (앱 진입 직후 말고 "첫 도전 선언 후" 등 맥락 있을 때 요청 → 허용률↑)
  - 🤖 Supabase에 푸시 토큰 저장 + 발송 트리거 (Edge Function / 스케줄)
  - 🧑‍💻 APNs(iOS) / FCM(Android) 인증서·키 설정 (개발자 계정 필요한 부분)
- [ ] **알림 정책**
  - 🧑‍💻 알림 종류·빈도 확정: 인증 리마인더, 인연 활동, "N명에게 용기를 줬어요" 등. 과다 알림은 이탈 유발 → 최소·고가치만
  - 🤖 유저별 알림 ON/OFF 설정 화면

- **`delete-account` Edge Function (service role + JWT 본인확인)**:
  - `public.users` 익명화: nickname→"탈퇴한 사람", avatar_url/email/google_sub→null, `deleted_at` set.
  - 개인 데이터 삭제: push 토큰·`user_interests`·`user_verifications`(0032)·알림함/큐 행.
